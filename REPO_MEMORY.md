# Repository Memory

This file serves as important collaborative memory for the current repository that can be synchronized via Git. Project-related experience, conventions, pitfalls, and test conclusions shall be prioritized and recorded here uniformly, instead of relying on temporary memory on personal devices.

## Memory Strategy

- Only record memory that has significant and long-term impact on the project to avoid information redundancy and increased maintenance costs caused by excessive recording.
- Record all project-level important memory only in this file, and synchronize it to Copilot's repository-level memory when necessary.
- Do not write repository facts into personal-level memory to avoid information drift during multi-device development.
- When adding new memory, prioritize recording executable facts: file locations, conventions, verified conclusions, regression risks, and test commands.
- Modify expired memory directly in this file without retaining historical versions to prevent misleading caused by outdated information.
